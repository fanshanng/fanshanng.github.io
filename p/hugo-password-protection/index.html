Hugo 博客添加文章密码保护功能

发布于2025-12-15
约8分钟，共1735字
0 次浏览
0 条评论

本文记录了如何为 Hugo 博客（基于 Stack 主题）添加文章密码保护功能。这是一个基于前端的轻量级保护方案，适用于静态博客。

实现原理

  1. 内容页保护：通过检查文章 Front Matter 中的 password 字段。如果存在，将正文内容进行 Base64 编码并隐藏。用户输入密码后，前端计算 SHA256 哈希值与预设密码比对，验证通过后解码并显示内容。
  2. 隐私防泄露：
    • RSS：在 RSS 订阅源中隐藏加密文章的正文，只显示提示信息。
    • 搜索：在站内搜索索引中排除加密文章的正文，防止通过搜索结果预览泄露内容。

修改步骤

主要逻辑是判断 .Params.password 是否存在。如果存在，则渲染一个密码输入框，并将真实内容加密存储在 JS 变量中。

<section class="article-content">
    <!-- 表格包装器逻辑保持不变 -->
    {{ $wrappedTable := printf "<div class=\"table-wrapper\">${1}</div>" }}
    
    {{ if .Params.password }}
        <!-- 密码输入框样式 -->
        <style>
            #article-lock {
                text-align: center;
                padding: 40px 20px;
                background: var(--card-background);
                border-radius: var(--card-border-radius);
                box-shadow: var(--shadow-l1);
                margin: 20px 0;
            }
            #article-lock input {
                padding: 10px;
                border: 1px solid #ddd;
                border-radius: 5px;
                margin-right: 10px;
                background: var(--body-background);
                color: var(--body-text-color);
            }
            #article-lock button {
                padding: 10px 20px;
                background-color: var(--accent-color);
                color: white;
                border: none;
                border-radius: 5px;
                cursor: pointer;
            }
        </style>
        
        <!-- 锁屏界面 -->
        <div id="article-lock">
            <h3>文章已加密</h3>
            <p>请输入密码查看内容</p>
            <div style="margin-top: 20px;">
                <input type="password" id="article-password" placeholder="Password">
                <button onclick="unlockArticle()">Unlock</button>
            </div>
            <p id="lock-error" style="color: red; display: none; margin-top: 10px;">密码错误</p>
        </div>
        
        <!-- 内容容器 -->
        <div id="article-content-protected" style="display: none;"></div>
        
        <!-- 解锁逻辑 -->
        <script>
            (function() {
                const correctHash = "{{ .Params.password | sha256 }}";
                // 对内容进行 Base64 编码
                const encodedContent = "{{ .Content | replaceRE "(<table>(?:.|\n)+?</table>)" $wrappedTable | base64Encode }}";
                const storageKey = "unlocked_" + window.location.pathname;

                async function sha256(message) {
                    const msgBuffer = new TextEncoder().encode(message);
                    const hashBuffer = await crypto.subtle.digest('SHA-256', msgBuffer);
                    const hashArray = Array.from(new Uint8Array(hashBuffer));
                    return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
                }

                function showContent(html) {
                    const container = document.getElementById('article-content-protected');
                    container.innerHTML = html;
                    container.style.display = 'block';
                    document.getElementById('article-lock').style.display = 'none';
                }

                function decodeContent() {
                    try {
                        const binaryString = atob(encodedContent);
                        const bytes = new Uint8Array(binaryString.length);
                        for (let i = 0; i < binaryString.length; i++) {
                            bytes[i] = binaryString.charCodeAt(i);
                        }
                        return new TextDecoder().decode(bytes);
                    } catch (e) {
                        console.error("Decoding failed", e);
                        return "";
                    }
                }

                window.unlockArticle = async function() {
                    const input = document.getElementById('article-password').value;
                    const inputHash = await sha256(input);
                    
                    if (inputHash === correctHash) {
                        const html = decodeContent();
                        showContent(html);
                        localStorage.setItem(storageKey, inputHash);
                        document.getElementById('lock-error').style.display = 'none';
                    } else {
                        document.getElementById('lock-error').style.display = 'block';
                    }
                };

                // 检查是否已解锁
                const savedHash = localStorage.getItem(storageKey);
                if (savedHash === correctHash) {
                    const html = decodeContent();
                    showContent(html);
                }
            })();
        </script>
    {{ else }}
        <!-- 普通文章直接显示 -->
        {{ .Content | replaceRE "(<table>(?:.|\n)+?</table>)" $wrappedTable | safeHTML }}
    {{ end }}
</section>

注意事项与安全性分析

安全性警告

是的，这种方式非常容易被破解。

请务必理解，这是一个前端保护方案，它的安全性非常有限，类似于"防君子不防小人"。

  1. Base64 不是加密：我们在模板中使用了 base64Encode 对内容进行编码。Base64 是一种编码方式，不是加密算法。任何懂一点技术的人，查看网页源代码，找到 encodedContent 变量，在浏览器控制台输入 atob(encodedContent) 即可直接看到文章明文。
  2. 内容已下发：虽然页面上看起来需要密码，但实际上文章的全部内容已经下载到了访客的浏览器中，只是被隐藏和编码了。
  3. 适用场景：此方案仅适用于防止普通访客直接查看内容，或者作为一种简单的访问门槛。绝对不要用这种方式保护银行账号、密码、私密照片等高度敏感信息。

改进思路（如果需要更高安全性）

如果需要真正的安全性，必须确保加密过程发生在构建阶段，且解密需要密钥（密码）。由于 Hugo 模板语言（Go Templates）本身不内置 AES 等加密函数，要实现真正的加密通常需要：

  1. 外部脚本预处理：编写 Python/Node.js 脚本，在 Hugo 构建前运行。脚本读取 Markdown 文件，使用密码对 content 进行 AES 加密，替换原文件内容。
  2. 前端解密：前端引入 crypto-js 库，用户输入密码作为密钥对密文进行解密。

目前的方案是 Hugo 原生支持的最简单方案，无需额外的构建步骤。

其他注意事项

  • HTTPS：crypto.subtle API 需要在 HTTPS 环境或 localhost 下才能运行。如果你的网站没有部署 HTTPS，解锁功能可能会失效。
